# PDF Encryption Research Report

**Date:** 2026-03-24 | **Researcher:** Claude Code
**Scope:** pdf-lib encryption capabilities, forks, alternatives, Cloudflare Workers compatibility

---

## Executive Summary

**pdf-lib v1.17.1 (official, unmaintained since ~2022):** NO encryption support. Multiple GitHub issues (#243, #233, #291, #1296, #1326) requesting feature—all closed as "not planned."

**Viable Paths:**
1. **pdf-lib-with-encrypt fork** (v1.2.1, last update Apr 2024) — ACTIVE, encryption API available
2. **@pdfsmaller/pdf-encrypt-lite** (7KB, RC4 128-bit) — PURPOSE-BUILT for Cloudflare Workers
3. **PDFKit** (Node.js library, v0.17.1) — Native encryption, NOT Cloudflare-native (requires nodejs_compat flag)

**Error When Loading Encrypted PDFs:**
```javascript
// Throws error:
PDFDocument.load(encryptedBytes);
// Error: Input document to PDFDocument.load is encrypted

// Workaround (does NOT work well):
PDFDocument.load(encryptedBytes, { ignoreEncryption: true });
// Result: blank/corrupted merged PDFs per issue #1390
```

**Conclusion:** Official pdf-lib is NOT viable for encryption. Do not use.

## Finding 2: pdf-lib-with-encrypt Fork

| Aspect | Detail |
|--------|--------|
| **Repository** | [sico/pdf-lib-with-encrypt](https://github.com/sico/pdf-lib-with-encrypt) |
| **npm Package** | `pdf-lib-with-encrypt` |
| **Latest Version** | 1.2.1 |
| **Last Commit** | April 24, 2024 |
| **Commits** | 524 total |
| **Maintenance Status** | ✅ ACTIVE (last update ~8 months ago) |
| **Encryption Support** | ✅ YES — dedicated "Encryption Handling" section in README |
| **npm Downloads** | ~1,336/week |

**Encryption API:**
- Exports standard `PDFDocument`, `StandardFonts`, `rgb()` from `'pdf-lib-with-encrypt'`
- Adds encryption dictionary support with `/Filter`, `/V`, `/CFM` entries
- Supports `/P` (user access permissions) — 32-bit integer flags for permissions
- Encryption methods: RC4 (v1.3 40-bit, v1.4 128-bit), AES (v1.7 128-bit, v1.7 ExtensionLevel 3 256-bit)

**Example (inferred from PDF spec + fork design):**
```javascript
import { PDFDocument } from 'pdf-lib-with-encrypt';

const doc = await PDFDocument.load(pdfBytes);
// Encryption setup via encryption dictionary
doc.encrypt({ userPassword: 'user123', ownerPassword: 'owner456' });
const encryptedPdf = await doc.save();
```

**Limitations:**
- No explicit API documentation in visible README—need to check `/docs` folder
- Smaller community than official pdf-lib
- May need to review code or test encryption dictionary structure

**Recommendation:** Best choice for Node.js/backend environments (Vercel, standard servers). Compatible with standard JavaScript environments. Test API on target project.

## Finding 4: Cloudflare Workers Solutions

### Option A: @pdfsmaller/pdf-encrypt-lite (RECOMMENDED)

| Aspect | Detail |
|--------|--------|
| **Package** | `@pdfsmaller/pdf-encrypt-lite` |
| **Size** | 7KB (ultra-lightweight) |
| **Encryption** | RC4 128-bit (standard PDF encryption) |
| **Cloudflare Compatible** | ✅ YES — designed for edge runtime |
| **Platforms** | Cloudflare Workers, Vercel Edge, Netlify Edge, browsers |
| **Repository** | GitHub (npmsearch mentions) |

**API:**
```javascript
import { encryptPDF } from '@pdfsmaller/pdf-encrypt-lite';

// With user password only
const encrypted = await encryptPDF(pdfBytes, 'user-password');

// With user + owner password
const encrypted = await encryptPDF(pdfBytes, 'user-password', 'owner-password');
```

**Installation:**
```bash
npm install @pdfsmaller/pdf-encrypt-lite pdf-lib
```

**Use Case:**
- Encrypt PDFs generated by pdf-lib in Cloudflare Workers
- Lightweight, no Node.js dependencies
- Works in browser or edge runtimes (V8)

**Key Advantage:** Designed specifically for serverless/edge environments. No Node.js polyfills needed.

### Option B: PDFKit (Node.js Library)

| Aspect | Detail |
|--------|--------|
| **Package** | `pdfkit` |
| **Version** | 0.17.1 |
| **Encryption** | ✅ YES — native support |
| **Cloudflare Compatible** | ⚠️ PARTIAL (requires `nodejs_compat` flag) |
| **npm Downloads** | Heavy use (enterprise library) |
| **Documentation** | [PDFKit Security Docs](https://pdfkit.org/docs/getting_started.html) |

**Encryption API:**
```javascript
const PDFDocument = require('pdfkit');
const doc = new PDFDocument({
  userPassword: 'user123',
  ownerPassword: 'owner456',
  permissions: {
    printing: 'highResolution',
    copying: false,
    modifying: false
  }
});
doc.pipe(fs.createWriteStream('encrypted.pdf'));
doc.text('Hello World');
doc.end();
```

**Password Constraints:**
- PDF 1.7 ExtensionLevel 3: max 127 bytes UTF-8
- Older versions: max 32 bytes, Latin-1 only

**Cloudflare Workers Compatibility:**
- Requires `nodejs_compat` compatibility flag (date: 2024-09-23+)
- Depends on Node.js APIs: `node:crypto`, `node:stream`, `node:buffer`
- Will work with flag enabled, but adds bloat vs. purpose-built solutions
- Better for standard backends than edge functions

**Key Disadvantage:** Not optimized for edge/serverless—adds overhead.

## Recommendation by Environment

### For Cloudflare Workers (V8 Runtime):
**Primary:** `@pdfsmaller/pdf-encrypt-lite` (designed for edge, 7KB, no polyfills)
**Secondary:** `pdf-lib-with-encrypt` (test for Workers compatibility; may work without nodejs_compat)

**Code Path:**
```javascript
import { PDFDocument } from 'pdf-lib';
import { encryptPDF } from '@pdfsmaller/pdf-encrypt-lite';

// Generate PDF with pdf-lib
const doc = await PDFDocument.create();
// ... add content ...
const pdfBytes = await doc.save();

// Encrypt with lightweight library
const encrypted = await encryptPDF(pdfBytes, 'password123');
return new Response(encrypted, {
  headers: { 'Content-Type': 'application/pdf' }
});
```

### For Node.js/Vercel/Standard Backend:
**Primary:** `pdf-lib-with-encrypt` (active maintenance, full encryption)
**Secondary:** `PDFKit` (larger but battle-tested, for PDF creation + encryption in one step)

### Avoid:
- Official `pdf-lib` (no encryption, unmaintained)
- `pdf-lib-plus-encrypt` (last update 3 years ago)

2. **@pdfsmaller/pdf-encrypt-lite Permissions:** Does this library support granular permissions (print, copy, modify flags)? Documentation unclear from npm page summary.

3. **Cloudflare Workers Actual Testing:** Neither pdf-lib-with-encrypt nor @pdfsmaller/pdf-encrypt-lite has explicit Cloudflare Workers case studies in public sources. Theoretical compatibility confirmed, but real-world testing needed.

4. **Browser Compatibility:** Can pdf-lib-with-encrypt or @pdfsmaller/pdf-encrypt-lite run in browser environments? Designed for Node.js vs. universal?
